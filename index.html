Draft protocol / reference implementation / formal model

Human approval, bound to the exact action.

HAPP gives relying parties portable evidence that a human explicitly approved a machine-readable action, with freshness, audience, replay, and presentation binding built in.

Read the draft Inspect implementations

Why

Authentication is not intent proof.

Agentic systems can be authenticated yet still execute the wrong thing. HAPP focuses on the missing control: verifiable proof that a human saw and approved this exact action now.

What gets bound

  • Intentintent_hash
  • What was shownpresentation_hash
  • Audience + freshnessaud, exp, jti
  • Challenge / replay statesingle-use verifier controls

Flow

Minimal relying party sequence

  1. RP blocks a sensitive action and returns a challenge.
  2. Agent requests approval from a HAPP provider.
  3. Human completes presence and consent in provider UI.
  4. Provider issues a signed consent credential.
  5. RP verifies hashes, audience, freshness, and replay state before execution.
